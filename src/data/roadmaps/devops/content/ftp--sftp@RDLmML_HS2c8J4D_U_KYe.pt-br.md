# FTP / SFTP

O protocolo FTP (File Transfer Protocol) é um protocolo de rede padrão usado para transferir arquivos entre um cliente e um servidor em uma rede de computadores. Ele opera em um modelo cliente-servidor, tipicamente usando conexões separadas de controle e dados entre o cliente e o servidor. O FTP permite que os usuários carreguem, baixem e gerenciem arquivos em sistemas remotos, suportando acesso autenticado e anônimo. Embora seja amplamente utilizado por sua simplicidade e compatibilidade, o FTP tem limitações de segurança, pois transmite dados e credenciais em texto simples. Como resultado, alternativas mais seguras como SFTP (SSH File Transfer Protocol) e FTPS (FTP Secure) ganharam popularidade para transferências de dados sensíveis. Apesar de sua idade, o FTP ainda é usado para várias necessidades de transferência de arquivos, especialmente em sistemas legados e onde a segurança não é uma prioridade.

O SFTP (SSH File Transfer Protocol) é um protocolo de transferência de arquivos seguro que fornece funcionalidades de acesso, transferência e gerenciamento de arquivos sobre um fluxo de dados seguro da shell SSH (SSH). Ele foi projetado como uma extensão do SSH para oferecer capacidades de transferência de arquivos seguras. O SFTP criptografa tanto os comandos quanto os dados em trânsito, protegendo contra ataques de escuta e "man-in-the-middle". Ao contrário do FTP tradicional, o SFTP usa uma única conexão e não separa os canais de controle e dados. Ele suporta recursos como retomada de transferências interrompidas, listagem de diretórios e remoção remota de arquivos. O SFTP é amplamente usado em ambientes corporativos para transferências de arquivos seguras, scripts automatizados e como uma alternativa mais segura ao FTP. Sua integração com o SSH o torna a escolha preferida para administradores de sistemas e desenvolvedores que trabalham com sistemas remotos de forma segura.
